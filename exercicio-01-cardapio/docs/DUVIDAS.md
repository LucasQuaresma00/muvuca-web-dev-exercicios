# ❓ Dúvidas comuns

## 📌 Por que usamos `select` ao invés de `input`?

Porque o `select` limita as opções do usuário, evitando valores inválidos e facilitando o controle da aplicação.

---

## 📌 Por que usamos `querySelector`?

Para selecionar elementos do HTML dentro do JavaScript de forma simples, utilizando seletores como `id` e `class`.

---

## 📌 Por que usamos `preventDefault()`?

Para evitar que o formulário recarregue a página ao ser enviado, permitindo manipular os dados com JavaScript.

---

## 📌 Por que usamos `display: flex`?

Para alinhar elementos horizontalmente de forma simples e moderna, sem precisar usar técnicas antigas como `float`.

---

## 📌 Por que usamos objetos para os preços?

Porque permite acessar os valores de forma direta (`precos[comida]`), deixando o código mais limpo e organizado.

---

## 📌 Por que usamos `.value`?

Para obter o valor selecionado ou digitado pelo usuário dentro dos campos do formulário.

---

## 📌 Por que usamos `addEventListener`?

Para escutar eventos do usuário, como o envio do formulário, e executar uma função quando esse evento acontece.

---

## 📌 Por que validamos os campos?

Para garantir que o usuário preencha corretamente os dados antes de processar o pedido, evitando erros na aplicação.

---

## 📌 O que é DOM?

DOM (Document Object Model) é a representação da página HTML no JavaScript, permitindo acessar e modificar elementos dinamicamente.

---

## 📌 Por que criamos elementos com JavaScript?

Para exibir o resultado dinamicamente sem precisar recarregar a página.

---

## 📌 Qual a diferença entre `id` e `class`?

* `id` → identifica um elemento único na página (usado principalmente no JavaScript)
* `class` → pode ser usada em vários elementos (usada principalmente no CSS)
